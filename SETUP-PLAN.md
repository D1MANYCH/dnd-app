# SETUP-PLAN

Промт-задание на настройку проекта. Сохранено как есть, для сверки в каждой фазе.

## Контекст проекта

D&D 5e PWA, репозиторий D1MANYCH/dnd-app.
Стек: vanilla JS (ES5/ES6), без сборщика и фреймворков.
Файлы: index.html, app.js (~4100 строк), data.js (~1230), style.css (~3950), spells.js
UI: русский, тёмная тема, mobile-first. Терминология D&D — русская, редакция 2014.
ОС: Windows, оболочка Git Bash: C:\Program Files\Git\bin\bash.exe
Ручные правки я делаю в Notepad++ — поэтому код должен быть модульным,
чтобы правки были точечными.

## Цель

Настроить проект так, чтобы разработку можно было вести с минимумом
ручной проверки с моей стороны. Работа разбита на фазы, каждая фаза —
отдельная сессия.

## Правила работы (действуют во всех фазах)

1. Сначала план: файл → функция → что меняем. До одобрения ничего не делай.
2. Триггер начала работы — моя фраза «начать фазу N». До неё только план.
3. Внутри фазы — по одному пункту, после каждого жди подтверждения.
4. Только точечные правки, файлы целиком не переписывай.
5. Комментарии в код не добавляй, сверх задачи ничего не рефактори.
6. Существующие файлы (включая CLAUDE.md) правь, а не создавай заново.
7. Каждая фаза — в отдельной ветке phase-N-имя. В конце фазы: превью,
   проверка в браузере, коммит.
8. Если что-то в задаче неоднозначно — спроси, не додумывай.

### ФАЗА 5 — rules.js + tests.html (ключевая фаза, не спеши)
- 5.1 Вынести из app.js в rules.js чистые расчёты, без обращений к DOM:
  модификаторы характеристик, бонус мастерства, владения и экспертиза,
  ячейки заклинаний по классам и уровням, спасброски, КД, инициатива,
  восстановление на коротком и длинном отдыхе.
  Поведение не меняем — только перенос, app.js работает через эти функции.
- 5.2 Создать tests.html: один самодостаточный файл, тёмная тема,
  жёлто-янтарные акценты, читаемый с телефона. Гоняет проверки
  функций rules.js и показывает зелёное/красное по каждой.
- 5.3 Покрыть проверками механики, которые уже ломались раньше:
  экспертиза Плута и Барда, Мастер на все руки, концентрация и
  спасбросок ТЕЛ при уроне, владения от предыстории, отдых,
  ячейки Колдуна.

Эталон — правила ДНД 2014. При любом сомнении в цифрах спрашивай,
не выдумывай.

### ФАЗА 6 — Разбиение app.js на модули
