- **~2 внешних вызова на запись:** 1 `WebSearch` на имя (отдаёт **оба** издания) + 1 `WebFetch` на читаемую страницу.
  Кэш URL делает поиск разовым на имя; ~318 имён общие для обеих редакций → экономия на поиске.
- **Батчить `WebFetch` 4-в-ряд** (как в пилоте) — кэш 15 мин/URL, запросов много (712× страниц).
- **Доверять только странице, не сводке поиска** (Mage Hand: поиск дал неверную школу).
- **Сверять фетч с каноном** при подозрении (Mage Hand: ложное «(с концентрацией)»).
- Данные БД на пилоте — **высокого качества**: 0 механических ошибок на 5 записях. Ожидаем, что основная
  работа фаз 1–6 — точечные правки + добавление алиасов для не покрытых имён, а не массовая чистка.
