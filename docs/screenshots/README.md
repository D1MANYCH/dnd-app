## Технические требования

- **Формат:** WebP (quality 82). Снимай в PNG, затем сконвертируй: `python tools/screenshots-to-webp.py` (Pillow) или `cwebp -q 82`. В репозиторий кладём только `.webp`.
- **Ширина:** 1200–1600 px (для retina лучше 1600+)
- **Тема:** тёмная (родная для приложения)
- **Содержимое:** заполненный персонаж 5–10 уровня (берём готовый билд)
- **Без хрома браузера:** скрин через DevTools (см. workflow ниже)

## Workflow для Windows (Chrome)

1. Открой https://d1manych.github.io/dnd-app/
2. `F12` → откроется DevTools
3. `Ctrl+Shift+M` → Device Mode
4. **Для десктопных скринов:** выбери «Responsive» вверху, задай размер `1440 × 900`
5. **Для мобильных:** выбери `iPhone 14 Pro` или `Pixel 7`
6. Создай персонажа или выбери готовый билд (5–10 уровень)
7. Открой нужный экран/модалку
8. `Ctrl+Shift+P` → начни вводить `screenshot` → выбери:
   - `Capture screenshot` — текущий вьюпорт
   - `Capture full size screenshot` — весь скроллящийся контент
9. Файл сохранится в `Downloads/` — переименуй по схеме выше и положи сюда

## Mockup-фрейминг (опционально, для канала)

Чтобы надеть скрин в рамку браузера или телефона:
- **shots.so** — самое красивое, бесплатно
- **screenshot.rocks** — попроще
- **mockuphone.com** — много моделей телефонов

Для README достаточно «голых» скринов без рамок — они компактнее в таблице.
