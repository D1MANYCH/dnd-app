# UI-план для v3.0 (фазы UI-1..UI-12)

**Why:** старый план R1-R7 закрыт в v2.6.0, блок UI-1..UI-3 из improvements_plan устарел после v3.0.0 (иконки PHB, гайды билдов, фильтры заклинаний переделаны). Нужен свежий набор UI-фаз поверх текущего состояния.

**How to apply:** триггер «начать фазу UI-N» — выполнить только эту фазу, верифицировать через preview_start + скриншот, остановиться. Рекомендованный порядок: UI-1 → UI-12 (но независимы, можно вразнобой).

## Фазы

### Темы и цвета
- ~~**UI-1. Светлая тема**~~ ✅ done — light v3 палитра cream (#eef2ef), warm radial gradient + SVG-noise, тогглер light/dark/auto, prefers-color-scheme через JS. Закрыто в v3.1.0 (commits 465c54d, 99d2dc6, fe5e29c).
- ~~**UI-2. Кастомизация акцента**~~ ✅ done в v3.3.0 — 8 пресетов (gold/emerald/ruby/amethyst/sapphire/copper/silver/graphite). CSS `:root[data-accent="..."]` + light-варианты. Превью-чипы 32px в drawer. Сохранение `dnd_accent`, применение до first paint в head.
- ~~**UI-3. Классовые акценты (auto-режим)**~~ ✅ done в v3.4.0 — `CLASS_ACCENT_MAP` (12 классов → 8 пресетов: Воин→silver, Жрец→gold, Друид→emerald, Колдун→amethyst и т.д.). Тоггл «🎭 Авто-акцент по классу» в drawer под чипами; в авто-режиме чипы dimmed (`is-disabled`). `_refreshAccent()` вызывается из `loadCharacter`, `showScreen('characters')` и `onchange` селекта класса. Хранение `dnd_auto_accent='1'`. Ручной клик по чипу выключает авто.

### Плотность и масштаб
- ~~**UI-4. Плотность UI**~~ ✅ done в v3.5.0 — три режима (compact/standard/cozy) через `:root[data-density]` с overrides `--sp-1..6`, `--fs-sm/md/lg/xl`, `--hp-bar-h`. Picker в drawer (3 кнопки). Хранение `dnd_density`, применение до first paint в head. `setDensity()` в app-ui.js.
- ~~**UI-5. Размер шрифта**~~ ✅ **done** в v3.7.1 — слайдер 90–130% (шаг 5%) с reset-кнопкой ↺. Механизм: `zoom` на `<body>` (на `<html>` сдвигает медиа-запросы и ломает desktop sidebar 1200px). Хранение `dnd_fs_scale`, применение через `DOMContentLoaded` (zoom на body нельзя ставить до парсинга body). Бонус: все настройки оформления (тема/акцент/плотность/масштаб) вынесены из drawer в отдельную модалку `#settings-modal` (Escape/✕/клик-вне закрывают) — drawer короткий, нав-пункты не обрезаются при cozy+130%. Адаптив ≤540px: кнопки заклинаний (Найти/Добавить) делят строку поровну с переносом текста.

### Навигация
- **UI-6. Мобильная навигация v2** — пересмотр таб-бара: безопасные зоны `env(safe-area-inset-bottom)`, увеличенные хитбоксы (≥44px), активный таб с индикатором-полосой, тактильная отдача через `navigator.vibrate(10)`. Свайпы влево/вправо между табами (Лист ↔ Заклинания ↔ Инвентарь ↔ Бой). Drawer с backdrop-blur и slide-from-left.
- ~~**UI-7. Адаптив для desktop/планшета**~~ ✅ done — sidebar 240px + main + right-rail 320px на ≥1200px, drawer-mode 1024-1199, tablet padding/grid-2 от 768px. Закрыто в v3.1.0–v3.1.5.

### Взаимодействия
- **UI-8. Drag-n-drop инвентаря** — перетаскивание предметов между слотами/категориями в [app-inventory.js](app-inventory.js) (используя готовый паттерн из [app-notes.js](app-notes.js)). Визуальный плейсхолдер «куда упадёт», подсветка целевой ячейки, отмена по Escape. На тач-устройствах — long-press 300ms активация.
- ~~**UI-9. Кнопка «Откатить уровень»**~~ ✅ done в v3.2.0 — снимок `char._prevLevelSnapshot` берётся в начале confirmLevelUp, откат через showConfirmModal с превью утрат (HP/умения), одношаговый undo. Кнопка показывается только если есть снимок.

### Визуальная полировка
- **UI-10. Skeleton-лоадеры и подсветка поиска** — skeleton-плейсхолдеры (мерцающие плашки `--bg-2 → --bg-3`) для списков заклинаний/предметов/билдов при первой загрузке. Подсветка совпадений `<mark>` в результатах поиска заклинаний/предметов с цветом `--accent-soft`.
- **UI-11. Анимации и микровзаимодействия** — fade+slide 150-200ms при смене вкладок, spring-анимация на нажатии кнопок (scale 0.96), pulse на HP-баре при уроне/лечении, count-up чисел (HP, опыт, золото). Уважать `prefers-reduced-motion`.
- **UI-12. Финальная полировка визуала** — пройтись по всем экранам с чек-листом: единые радиусы (`--r-md` для карточек, `--r-pill` для чипов), консистентные тени, выравнивание по 4px-сетке, отступы между секциями `--sp-5`. Тёмная/светлая прогон-проверка скриншотами всех 7 вкладок + 5 ключевых модалок (level-up, бросок кубика, выбор подкласса, билд-пикер, заклинание-карточка). Удаление мёртвых стилей (`style.css` 8224 строк — почистить).
