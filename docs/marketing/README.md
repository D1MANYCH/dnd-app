# Контент-план DnD-Листа

Редакционный материал для Telegram-канала [@dndlistru](https://t.me/dndlistru) и посевов в сообществах. Не пользовательская документация — это рабочий черновик.

## Ритм публикаций

| День | Тип поста | Источник контента | Время на пост |
|---|---|---|---|
| Понедельник | **Билд недели** | `character-builds.js` + `build-notes-data.js` (40 билдов = 40 недель) | 30–40 мин |
| Вторник | Фича-разбор | Скрин/GIF неочевидной фичи | 20 мин |
| Среда | Опрос / engagement | — | 5 мин |
| Четверг | Гайд по заклинаниям | `spells.js` (712 заклинаний) | 20–30 мин |
| Пятница | Релиз-нота (если был bump) или подборка | `APP_CHANGELOG` или сводка | 5–10 мин |

Не каждую неделю все 5 — норма 3–4 поста/неделя, чтобы не выгореть.

## Принципы

- **Польза в первой строке.** Никаких «всем привет, сегодня я расскажу про…»
- **Скриншоты, не текст-стены.** Из 6 канон-скринов в `docs/screenshots/` + бонусы из `docs/screenshots/bonus/`
- **Эмодзи как навигация**, не как декор. 1–2 в заголовке, ⚪ ✅ ❌ для маркеров — норм
- **Ссылка одна** — на приложение `d1manych.github.io/dnd-app`. На GitHub/донаты — только в специальных постах, не в каждом
- **Без призывов «подписаться»** в первые недели. Сначала ценность, подписка — следствие
