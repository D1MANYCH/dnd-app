# Контент-план DnD-Листа

Редакционный материал для Telegram-канала [@dndlistru](https://t.me/dndlistru) и посевов в сообществах. Не пользовательская документация — это рабочий черновик.

## Позиционирование: «Разраб-практик»

Бренд канала — **игрок D&D, который сделал инструмент для себя, делится с другими**. Не мастер, не оптимизатор, не контент-мейкер. Контент-посты — от первого лица, в личной интонации. Исключение — **релиз-посты**: они нейтральные и фактические, без личного голоса (см. «Два жанра постов»).

**Что это значит на практике:**

- Пишу «я добавил», «у меня была сессия», «заметил баг» — не «вы должны», «оптимальный билд — это»
- Не претендую на экспертизу по тактике/правилам D&D. Если спрашивают сложное про оптимизацию — переадресую в чат: «в обсуждении есть знатоки»
- Контент строится вокруг **приложения**, а не вокруг правил D&D
- Спорные DnD-темы (оптимизация билдов, метагейминг) — не моя поляна, не лезу

**80% постов — про приложение** (где автор эксперт), **20% — личные наблюдения игрока** («заметил странность в правилах», «вот так настроил свой стол»).

## Ритм публикаций

| День | Тип поста | Источник контента |
|---|---|---|
| Понедельник | **Что нового / что планирую** | твой changelog + roadmap |
| Среда | Фича-разбор или опрос | скрин/GIF фичи или вопрос аудитории |
| Пятница | Личная заметка или релиз-нота | твой опыт игрока + APP_CHANGELOG |

Норма 3 поста/неделя — устойчиво, не выгораешь. Можно меньше — без угрызений совести.

## Принципы

- **Польза или личное наблюдение в первой строке.** Никаких «всем привет»
- **Скриншоты, не текст-стены.** 6 канон-скринов в `docs/screenshots/` + бонусы
- **Эмодзи как навигация**, не как декор
- **Ссылка одна** — на приложение `d1manych.github.io/dnd-app`. На донаты/GitHub — только в специальных постах
- **Чат-обсуждение** в каждом посте, где призываешь к диалогу (баги, идеи, опросы)

## Два жанра постов

**1. Релиз-посты — нейтральные, фактические.** Выход новой версии. Без личного голоса, сухой факт «вышло обновление». Шаблон:

```
📦 DnD-Лист vX.Y.Z · DD месяца ГГГГ

[1–2 фактических предложения: что добавлено/изменено]

Главное:
[эмодзи] ключевой пункт
[эмодзи] ключевой пункт
[эмодзи] ключевой пункт   (3–5 пунктов максимум)

📋 Полный список изменений → github.com/D1MANYCH/dnd-app/blob/main/CHANGELOG.md#vX.Y.Z
🎲 d1manych.github.io/dnd-app
💬 t.me/DnDSocialru
```

Полный список изменений всегда уходит в `CHANGELOG.md` по якорю версии — в посте только 3–5 главных пунктов. Готовые примеры: `posts/06-stabilizaciya.txt`, `posts/07-pdf-export.txt`, `posts/08-srd-bestiary.txt`. Личные версии этих же релизов (старый стиль) — в `posts/archive/`.

**2. Контент-посты — личный голос «разраб-практик».** Истории, фичи, опросы, FAQ. От первого лица. Примеры: `posts/01-05`.

### CHANGELOG.md

Файл генерируется из `APP_CHANGELOG` в `data.js`:
```
node tools/gen-changelog.js
```
Перезаписывается целиком — вручную не править. Прогонять после каждого релиза (`/bump`) и коммитить вместе с ним. Ссылка для релиз-постов: `github.com/D1MANYCH/dnd-app/blob/main/CHANGELOG.md#v<версия>`.

## Чего я НЕ публикую

- Гайды по оптимизации билдов (не моя экспертиза)
- Детальные разборы тактики D&D (не моя экспертиза)
- «Топ-5 лучших…» — авторитетные подборки требуют авторитета
- Споры на тему правил RAW vs RAI

Если хочется такого контента — позже найду опытных игроков в обсуждении, попрошу написать гостевой пост с честной атрибуцией («написал @nickname из чата»).

## Два канала коммуникации

**Канал [@dndlistru](https://t.me/dndlistru)** — дисциплинированный контент по графику (3 поста/неделю). Релизы, фича-разборы, опросы, FAQ. Полноценные посты со скринами.

**Чат-обсуждение** ([@DnDSocialru](https://t.me/DnDSocialru)) — неформальная кухня, привязана к каналу как discussion group. Под каждым постом канала кнопка «Прокомментировать» ведёт в комменты к этому посту. В чате — мини-апдейты, work-in-progress, ответы на вопросы, баг-репорты. Темп: «когда есть что сказать», не по графику.

Подробно про контент в группе — [group-content.md](group-content.md).
Закреп для группы — [group-pinned.txt](group-pinned.txt).
