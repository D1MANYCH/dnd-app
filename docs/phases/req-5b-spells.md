# REQ-5b: Названия заклинаний по dnd.su

Триггер для нового чата: **«начать фазу REQ-5b»** (или открыть этот файл и сказать «выполни»).

## Context
Часть аудита названий (см. [docs/naming-audit.md](../naming-audit.md)). Эталон — **dnd.su** (решение пользователя). REQ-5a (навыки/раса/язык/предыстории/состояния) уже выпущен (v3.28.24–27). Здесь — **заклинания**.

Витрина заклинаний (`spells.js`, поле `name`) переведена в стиле **Fantom Studio**; dnd.su использует другие названия. Прошлые SPELL-AUDIT-фазы сверяли только **данные** (школы, дистанции, концентрация) — имена не трогали. Объём — 706 заклинаний.

## Готовая основа
Карта `_SPELL_ALIASES` в [app-core.js](../../app-core.js) (искать `_SPELL_ALIASES`) уже фиксирует пары **dnd.su-имя → имя БД** (для разбора билдов). Её нужно **инвертировать** в карту переименований БД→dnd.su. Документировано ~20 пар, например:

| Сейчас в БД (витрина) | dnd.su (цель) |
|---|---|
| Огненный болт | Огненный снаряд |
| Луч мороза | Луч холода |
| Волшебная рука | Рука мага |
| Священный огонь | Священное пламя |
| Руководство | Указание |
| Сон | Сонливость |
| Громовая волна | Удар грома |
| Дубина | Дубинка |
| Дружелюбие | Дружба |
| Злобная насмешка | Насмешка |
| Метка охотника | Охотничья метка |
| Мнимая жизнь | Ложная жизнь |
| Истинное зрение | Истинное видение |
| Силовая клетка | Узилище |
| Оцепенение | Помутнение разума |

(+ остальные в `_SPELL_ALIASES`; и по всем 706 — вероятно десятки недокументированных, нужен системный проход.)

## Риск и стратегия (важно)
Имена заклинаний хранятся **по значению** в сохранёнках: `char.spells.prepared` — массив имён (см. REQ-2/BUILD-FIX-13), `char.spells.mySpells` — объекты с `name`. Переименование витрины **сломает привязку**, если не предусмотреть совместимость.

**Безопасный план:**
1. Переименовать поле `name` в `spells.js` на dnd.su. **Партиями** (по уровню заклинаний 0→9 или по школам), чтобы каждый шаг верифицировать.
2. В `_SPELL_ALIASES` оставить/добавить пары так, чтобы и **старые Fantom-имена**, и dnd.su-имена резолвились (билды уже шлют dnd.su-имена — после переименования совпадут напрямую; старые Fantom-имена остаются алиасами).
3. **Миграция сохранёнок** (новый `if (v < N)` в `migrateCharacter`, бамп `SCHEMA_VERSION` в data.js): прогнать `char.spells.prepared` и `char.spells.mySpells[].name` через карту Fantom→dnd.su. Обновить ассерты схемы в `tests/headless.js` (там хардкод текущей версии).
4. Проверить инлайн-описания заклинаний на упоминания переименованных (как было с навыком «(Анализ)» в `spells.js`).

## Где править
- `spells.js` — поле `name` (партиями).
- `app-core.js` — `_SPELL_ALIASES` + новый блок миграции `migrateCharacter`.
- `data.js` — `SCHEMA_VERSION` (+1).
- `tests/headless.js` — ассерты `schemaVersion` под новую версию.

## Verify (после каждой партии)
- `node tests/headless-node.js` — зелёный.
- `verifyAllBuilds()` в браузере (подгрузить `dev-verify-builds.js`) — **36/36 fullPass**.
- Preview: применить билд → заклинания в витрине показывают dnd.su-имена; отметка «подготовлено» (`prepared`) совпадает с `mySpells` (0 «сирот»).
- Миграция: легаси-чар с `prepared:["Огненный болт"]` → после `migrateCharacter` имя dnd.su, привязка к заклинанию работает.
- SW-кэш при проверке shipped: unregister SW + clearCaches + reload (см. [reference в naming-audit]).
