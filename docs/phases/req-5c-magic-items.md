# REQ-5c: Названия магических предметов по dnd.su

Триггер для нового чата: **«начать фазу REQ-5c»** (или открыть этот файл и сказать «выполни»).

## Context
Часть аудита названий (см. [docs/naming-audit.md](../naming-audit.md)). Эталон — **dnd.su**. REQ-5a выпущен (v3.28.24–27). Здесь — **каталог магических предметов** `magic-items.js` (193 позиции), созданный в REQ-4a с распространённым переводом + английским именем (`nameEn`). Нужна сверка русских названий с dnd.su.

## Риск — низкий
- `magic-items.js` — свежие данные, **не хранятся в сохранёнках по ссылке**: при выборе предмета `fillFromMagicItem` (app-inventory.js) **копирует** название в инвентарь. Переименование каталога **не ломает** уже добавленные предметы и не требует миграции.
- Каталог грузится лениво (`ensureMagicItems`), английское имя (`nameEn`) остаётся для поиска. Сборку/тесты/билды не затрагивает.

## Где править
- `magic-items.js` — поле `name` у записей `MAGIC_ITEMS` (193 шт.). `nameEn`, `rarity`, `type` не трогать.
- Возможные точечные правки в описаниях (`desc`), если есть расхождения терминов.

## Что сверить (примеры спорных)
Пройти все 193 названия по dnd.su. Особое внимание к неустоявшимся переводам, например:
- Vorpal Sword — сейчас «Вострый меч» → проверить dnd.su.
- Flame Tongue, Frost Brand, Holy Avenger, Sun Blade, Deck of Many Things, Bag of Holding, Ioun Stone и т.п. — сверить с dnd.su.
- Зелья силы великанов / пояса силы великанов — единообразие с расой/языком из REQ-5a (холмовых/каменных/огненных/облачных/штормовых).

Метод: для каждого предмета `nameEn` → найти на dnd.su → выставить русское `name`. Где dnd.su расходится с офиц. переводом — берём dnd.su.
