# REQ-5f — Механический проход (descs/полнота умений)

**Эталон:** книга **PHB 2014 «PHantom»** (`../DND rules/DnD5e Книга Игрока.pdf`), сверка по факту текста через pymupdf (13.0pt=базовое умение класса, 11.0pt=умение подкласса). См. [[reference_dnd_rulebooks_pdfs]].

**Цель:** закрыть накопленные за REQ-5d/5e `flag (content)` — описания умений (`feature.desc`), которые НЕ соответствуют книжной механике уровня (выдуманы / смещены по уровням / перепутаны), и пробелы (умение книги ОТСУТСТВУЕТ в app). Имена уже сведены к книге в REQ-5d/5e — здесь правится механика отображения.

## Область и безопасность
- В scope **только PHB-подклассы и базовые классы** (как REQ-5d/5e). Подклассы из Tasha's/Xanathar's/DMG/SCAG — вне эталона, не трогаем.
- `feature.name`/`feature.desc` — **только отображение** (`updateClassFeatures` в `app-combat.js`, `openClassPlan` в `app-core.js`). Ключи = класс/подкласс + уровень. **desc-правки миграции НЕ требуют.**
- **Две категории находок:**
  - **D (desc-only)** — переписать текст `feature.desc` под книжную механику. Низкий риск, без миграции.
  - **F (functional)** — умение книги ОТСУТСТВУЕТ или смещено по уровню; нужно добавить/переместить запись, иногда ресурс/выбор. Может затронуть `CLASS_RESOURCES`, `*_CHOICES`, билды, leveling, тесты. **Каждый F-флаг — отдельное решение в момент партии** (добавлять механику vs только desc). Риск выше — проверять `verifyAllBuilds()` + `verifyAllLevelUpData()` + preview.
- **Не выдумывать механику** сверх книги. Где app-desc неполон, но не противоречит — оставлять (пометить). Прецедент рецепта REQ-5c (баг Ioun-Fortitude оставлен как out-of-scope).
- Каждая партия = патч-релиз (`/bump` → `/preflight` → коммит по запросу).

## Метод сверки
В момент партии — извлечь текст соответствующих страниц книги (pymupdf, UTF-8 в файл, не в stdout cp1251), сверить app-desc с книжной механикой, классифицировать D/F, применить только подтверждённое.
