# Аудит названий (REQ-5)

Жалоба: «Названия отличаются от привычных как из официальных книг или с dnd.su».

**Причина.** Базовые наборы переведены по **Fantom Studio PHB 5e** (явно помечено в `data.js`: расы, предыстории, навыки, черты). dnd.su и официальный перевод (Hobby World) местами используют другие термины. Заклинания ранее сверяли с dnd.su только по **данным** (школы, дистанции, концентрация) — **названия не приводили**.

**Источник истины (предлагается):** dnd.su как первичный (самый ходовой в RU-сообществе). Где dnd.su и офиц. перевод расходятся — помечено.

> **Статус (19.06.2026):** эталон = **dnd.su** (решение пользователя). **REQ-5a завершён** — `verifyAllBuilds` 36/36, тесты 242/242 на каждом патче:
> - **Навыки** (v3.28.24): Аркана→Магия, Расследование→Анализ.
> - **Раса/язык** (v3.28.25): Дварф→Дворф, Дварфский→**Дворфийский**; миграция (schema 13).
> - **Предыстории** (v3.28.26): Воин→Солдат, Благородный→Дворянин, Герой народа→Народный герой, Матрос→Моряк, Торговец/Подмастерье→Гильдейский ремесленник; слиты дубли Аколит/Преступник-Шпион/Бродяга (15→11); миграция (schema 14).
> - **Состояния** (v3.28.27): прилагательные формы dnd.su (Ослеплён→Ослеплённый и т.д.), id не тронуты.
>
> **Осталось (по запросу):** 5b заклинания (706) и 5c маг.предметы (193) — сверка имён с dnd.su.

## 1. Навыки (18) — почти всё ок, 2 кандидата
| В приложении | dnd.su | Прим. |
|---|---|---|
| Аркана | **Магия** | dnd.su переводит Arcana как «Магия» |

> Риск низкий: навык хранится по индексу (`skills[i]`), не по имени → переименование безопасно (сохранёнки не ломаются).

## 2. Раса «Дварф» → «Дворф» (+ внутренняя нестыковка) 🔴
- Расы в `index.html`: **«Дварф»**, «Горный дварф», «Холмовой дварф».
- Но в `CLASS_FEATURES`/`SUBCLASS_FEATURES` и в новом `magic-items.js` уже используется **«дворф/Дворфийский»**.
- dnd.su стандарт — **«Дворф»**.

→ Привести расы к «Дворф» (и зависимые строки: `RACE_*`, опции select). **Внутренняя нестыковка приложения сама по себе баг.**
Остальные расы совпадают с dnd.su (Тифлинг, Драконорождённый, Аасимар, Голиаф, Полурослik и т.д.).

> Риск средний: раса хранится по строке (`char.race`) и используется ключом в `RACE_LANGUAGES`/`RACE_ARMOR`/билдах. Нужна миграция сохранёнок + алиас «Дварф»→«Дворф».

`BACKGROUND_SKILLS` (data.js) использует Fantom-названия; билды уже шлют PHB/dnd.su-имена через алиас-карту `_bgAliases` (app-core.js) — т.е. расхождение уже признано в коде. Кандидаты:

| В приложении (ключ) | PHB/dnd.su | Англ. |
|---|---|---|
| Воин | **Солдат** | Soldier |
| Благородный | **Дворянин** | Noble |
| Подмастерье | **Гильдейский ремесленник** | Guild Artisan |
| Чужеземец | **Чужеземец / Дикарь** | Outlander |
| Матрос | **Моряк** | Sailor |
| Прислужник / Аколит | **Прислужник** | Acolyte (есть дубль!) |
| Преступник/Шпион | **Преступник** | Criminal/Spy |

→ Привести ключи `BACKGROUND_SKILLS` к dnd.su, убрать дубль Прислужник/Аколит, обновить `_bgAliases` (оставить старые как алиасы).
