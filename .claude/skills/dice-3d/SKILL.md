# Дайс-подсистема: архитектура и известные грабли

## Слои (сверху вниз)

1. **`quickRoll({label, sides, mod, mode})`** — `app-ui.js` (~строка 476, искать
   `function quickRoll`). Универсальная точка входа для ЛЮБОГО нового источника бросков
   (лист, бой, лента). Пишет в общую историю с подписью, обновляет ленту `#quick-roll-strip`.
   Новые фичи бросков строить поверх неё, не поверх animateDice3d напрямую.
2. **`animateDice3d(sides, result, callback, opts)`** — `app-ui.js` (~1328). Контракт
   `callback(v1, v2)` — на него завязан `app-inventory.js`, НЕ менять сигнатуру.
3. **`vendor/dice-box/`** — вендоренный @3d-dice/dice-box (WebGL/Babylon.js в воркере).

## ⚠️ Гонка бросков — решена, не ломать (DICEFIX, v3.25.4)

Корневые факты о вендоренной библиотеке:
- `box.roll()` **сам вызывает `clear()`** внутри — параллельные roll() библиотекой
  не поддерживаются в принципе.
- `box.roll()` возвращает **per-roll промис** с результатами именно этого броска —
  используется он, а НЕ перезаписываемый `onRollComplete` (на него не переходить).

Решение — interrupt-семантика в `app-ui.js` через module-var `_dice3dActiveRoll` (~626):
- новый бросок мгновенно финализирует предыдущий его precomputed-результатом;
- таймаут действует только на текущий бросок;
- guard `roll.done` перед `box.roll()`.

При любом дайс-баге сначала проверять, не нарушена ли эта семантика.

## Верификация бросков

- Физика в СКРЫТОЙ вкладке preview не идёт (rAF заморожен) — вкладка должна быть видимой.
- `preview_screenshot` виснет из-за WebGL — см. скилл `verify-ui`.
- Результат броска детерминирован заранее (precomputed), 3D — только анимация: логику
  можно тестировать headless (`tests/headless-node.js`), анимацию — только глазами/кликом.
