# Добавление контента: файлы, эталон, верификация

## Эталон названий и механики

Русские официальные книги 2014 (аудит REQ-5 завершён в v3.28.63 по книгам: навыки, расы,
предыстории, состояния, 706 заклинаний, 193 магпредмета, фичи подклассов, базы 12 классов).
НЕ выдумывать переводы — если название спорное, спросить пользователя или найти соседнюю
запись с тем же источником. Для контента редакции 2024 — план E24 (стартует после FIN).

| Глоссарий-тултипы | `glossary-data.js` (`glossarizeHtml`/`.gloss`) | preview |

## Миграции персонажа (schemaVersion)

Если новое поле нужно в СОХРАНЁННЫХ персонажах — добавить шаг миграции в `app-core.js`
(цепочка `char.schemaVersion` начинается ~строка 231; искать `schemaVersion`).
Правила: новый шаг строго `N → N+1` в конец цепочки, старые шаги не трогать,
дефолт для отсутствующего поля. Тесты миграций — в `tests/headless-node.js`.

## Чек-лист сдачи

1. `node tests/headless-node.js` зелёный (хук прогонит сам, но читать вывод).
2. Для билдов — `verifyAllBuilds()` 36/36.
3. Если менялись shipped-файлы и нужен релиз — скилл `release` (инвариант версий).
4. В preview проверять с учётом ловушки SW-кеша — скилл `verify-ui`.

## Что НЕ делать

- Не подключать сборщик/npm-runtime — всё vanilla JS.
- Не менять `assets/` без bump `CACHE_NAME`.
- Не «улучшать» механику от себя — только по книгам или по явному запросу.
